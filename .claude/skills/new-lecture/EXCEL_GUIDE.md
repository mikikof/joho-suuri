# Excel 演習ワークブック規約集

lec03 で確立した Excel (`.xlsx`) 演習シートの規約。**ターゲット環境は Google スプレッドシート**（Excel ではない）。新しい回でもこの規約に従う。

---

## ファイル名と保存場所

```
lecNN/情報数理入門_第N回.xlsx
```

半角アンダースコア + 全角「第N回」。

---

## シート構成 (10 シート)

```
1_<PART1テーマ>           PART 1 と 1 対 1 対応
2_<PART1-2テーマ>         PART 1-2 がある場合は別シート
3_<PART2テーマ>           ※ シート番号は連番、PART 番号と一致するわけではない
…
N_<PARTNテーマ>
演習①_基本               HTML より一段レベルアップ、表を埋めて可視化
演習②_応用               多次元・複合条件、表を埋めて可視化
挑戦①_<テーマ>           発展題材、表を埋めて可視化
挑戦②_<テーマ>           試行錯誤型、入力セルを動かして最適化
```

シート名は短く具体的に（lec03 例: `1_座標` `2_リスクリターン` `3_距離` `4_円` `5_領域` `6_類似度` `挑戦①_2 種類の距離` `挑戦②_最適立地`）。

---

## レイアウトの統一型

各シート（PART 別 / 演習・挑戦 共通）は同じ縦のレイアウトに揃える:

```
B 列〜             右側（H 列 or G 列など）
─────────────────────────────────
[タイトル]         （何もなし）
[ストーリー]       （何もなし）
[表ヘッダ]         [▶ アクション枠 1]
[表データ ...]     [▶ アクション枠 2]
[結果セル]         [▶ アクション枠 3]
```

- **左ブロック**: 表とデータ（黄色セル含む）
- **右ブロック**: 黄色のアクション枠で操作指示

---

## 黄色アクション枠（最重要パターン）

操作指示はすべて「**黄色のアクション枠**」で目立たせる。**ラベルは厳格に 3 種類のみ**:

| ラベル(厳守) | 用途 |
|---|---|
| **▶ 黄色を埋める** | サンプル式が入っている → ドラッグで広げる手順 |
| **▶ グラフで可視化** | データ範囲を指定して散布図/縦棒を作成する手順 |
| **▶ 動かしてみる** | 入力セルを変えて結果が動くのを観察する手順 |

> **注意 (lec04 codex 監査での発見)**: 上記 3 種以外のラベル(例: `▶ 頂点を観察` `▶ 一行 Python の力` `▶ 機械の答えと比べる` `▶ 読み取り` `▶ 読み取り + 試行` `▶ 黄色を埋める + グラフ` `▶ 機械が線を引く`)を発明しないこと。観察・読み取り・比較・試行はすべて `▶ 動かしてみる` に集約する。グラフ作成と黄色を埋める手順を一緒に書きたい場合は **アクション枠を 2 つに分ける**(まとめない)。

枠の見た目（openpyxl 実装）:

```python
def action_box(ws, tl, br, label, body):
    """tl の行をラベル行、その下〜br までを本文行とする黄色枠"""
    # ラベル行: 濃いめの黄色 (FFFDE68A) + 太字オレンジ (FF92400E)
    ws[tl] = f'▶  {label}'
    ws[tl].font = Font(name='Yu Gothic', size=12, bold=True, color='FF92400E')
    ws[tl].fill = PatternFill('solid', fgColor='FFFDE68A')
    ws.merge_cells(f'{tl}:<ラベル行末>')
    # 本文: 薄い黄色 (FFFEF3C7)
    body_top = f'{tl の列}{tl の行+1}'
    ws[body_top] = body
    ws[body_top].font = Font(name='Yu Gothic', size=11, color='FF422006')
    ws[body_top].fill = PatternFill('solid', fgColor='FFFEF3C7')
    ws.merge_cells(f'{body_top}:{br}')
```

各シートに **3 個前後** のアクション枠を置く（多すぎると埋もれる）。

## 黄色セル = 編集対象セル

- **黄色 (FFFEF3C7)** のセル = 生徒が触る対象（式入力 / 数値入力）
- 結果セル（最短距離・件数など）も同じ黄色（生徒は触らないが、式が入って結果が出る）
- **サンプル式は 1 行目だけ**入れて、残りの行は空欄黄色にする → 「ドラッグで広げる」体験を必ず作る

例（演習①_基本 Q1）:

```
E15: =SQRT(C15^2+D15^2)   ← サンプル式（黄色）
E16: 空欄                  ← 黄色のまま（生徒がドラッグで埋める）
E17: 空欄                  ← 同上
…
```

## 「問い」「予想」「ヒント」「発展」は使わない

ユーザー方針 (lec03 で確立):

- 演習を「**表を関数で埋めて、グラフで可視化する**」のシンプルな型に揃える
- 「【問い】」「【予想】」「【ヒント】」「【発展】」のような考えさせる節は**削除**
- 操作指示は黄色アクション枠だけで完結させる
- 演習・挑戦シートでも同じ。「問い」を残さない

### 禁止語と代替例

| 禁止語 | 代替例 |
|---|---|
| 「ヒント」 | 「目安」「参考に」「を踏まえて」 |
| 「予想」 | (削除、 単に作業させる) |
| 「問い」 | 「Q1 / Q2」 のような番号だけにする |
| 「発展」 | 「挑戦①」「挑戦②」 のシート分割で対応 |

## Google スプレッドシート前提の操作用語

すべての操作手順は Sheets の UI 用語で書く（Excel ではない）:

| ✗ Excel 寄り | ✓ Sheets 寄り |
|---|---|
| 散布図（直線あり） | 散布図を作成 → ダブルクリック → [カスタマイズ] → [系列] → **「線の太さ」を 0px から 2px 以上に上げる**(Sheets には「データポイントを線で結ぶ」という日本語チェックボックスは **存在しない**。「線の太さ」 を変えるか、グラフ種類自体を「折れ線グラフ」に変更する)|
| グラフ右クリック → データの選択 → 系列追加 | **(避ける)** Sheets の「系列を追加」は X 軸が共通の追加用。X 軸が違う 2 系列(実測 vs 予測)を入れたい場合は **最初から複数列(B:D など)で散布図にする**のが確実。Sheets が列単位で自動的に系列を分離する |
| グラフの軸の最小最大を固定 | ダブルクリック → [カスタマイズ] → [横軸/縦軸] → 最小値・最大値 |
| セル右下の■をドラッグ | セル右下の **フィルハンドル**（青い四角）をドラッグ |
| フィルター | メニュー [データ] → [フィルタを作成] |
| スピンボタン / スクロールバー | **(使わない)** ActiveX は Sheets で動作しない。普通の数値入力セル(黄色)＋「動かしてみる」アクション枠で代替 |

### Google Sheets で散布図を作るときの落とし穴

- **デフォルトは縦棒グラフ**。「グラフの種類を散布図に変更」を必ず明示
- **データ範囲は数値列だけ**(名前列を含めると X 軸が文字列カテゴリになって座標として機能しない)
  - ✓ `C9:D15` (x 列 + y 列)
  - ✗ `B9:D15` (名前 + x + y)
- 名前ラベルを点に出したい場合は別途 [カスタマイズ] → [系列] → 「データラベル」 → ラベルの範囲を指定

### ⚠ lec04 の試行錯誤で発見した落とし穴

- **「データポイントを線で結ぶ」 という日本語 UI は Sheets に存在しない**
  - lec03 EXCEL_GUIDE 旧版にこの誤りがあったが、実機で確認できない
  - 正しい操作: カスタマイズ → 系列 → **「線の太さ」 を 0px から 2px 以上に上げる**(これで点が線で結ばれる)
  - フォールバック: グラフ種類を **「折れ線グラフ」** に変える(X 軸が等間隔の数値ならそのまま動く)

- **「系列を追加」 ボタンは X 軸を共通にした追加用**
  - X 軸が違う 2 系列(例: 実測 + 予測直線)を 1 グラフに入れたい場合、Sheets の「系列を追加」 では迷う
  - **正解: 最初から複数列で散布図** にする
    - 例: B 列(x) + C 列(実測 y) + D 列(予測 y) を `B9:D14` として選択 → 散布図 → C と D が自動で 2 系列になる
    - その後カスタマイズ → 系列 → 「予測」 系列だけ「線の太さ」を 2px に上げる

- **不連続範囲指定はユーザーが迷う**
  - ✗ 「`B9, D9:D14` を選択」 → Cmd-クリック必須、説明不足で迷子
  - ✓ 「`B9:D14` を選択」 → 連続範囲なら通常選択でできる
  - どうしても不連続が必要なら「Cmd-クリック(Mac) / Ctrl-クリック(Win)で複数範囲選択」 と明記

- **散布図の点を線で結ぶ際、X 軸の昇順性が必須**
  - 線で結ばれる順序は **データの並び順**(x 値ではない)
  - 表でデータを書く時は x 昇順で並べないと、線がジグザグになる

- **Sheets で動かない / 動作が違う関数(避ける)**
  - `IFS` `XLOOKUP` `LAMBDA` `LET` `FILTER` などの新しい関数 → 古い Sheets で `#NAME?` エラー
  - VBA マクロ → Sheets では Apps Script に置き換えが必要、配布教材では使わない

## 数式の基本セット

両環境で動く関数だけ使う:

### 基本算術・条件分岐
- `=SQRT(...)` ／ `^` (二乗)
- `=IF(条件, 真, 偽)` (入れ子可)
- `=COUNTIF(範囲, "値")`
- `=MIN(範囲)` / `=MAX(範囲)` / `=AVERAGE(範囲)` / `=SUM(範囲)` / `=SUMPRODUCT(範囲1, 範囲2)`
- `=INDEX(範囲, MATCH(値, 範囲, 0))` (最近傍探索 / 最大値の場所探索の定石)
- `=ABS(...)` (マンハッタン距離 / 残差の絶対値)
- `=CONCATENATE(...)` (文字列連結、方程式の表示など)
- 絶対参照 `$C$10` (ドラッグで参照を固定)

### 統計・回帰関数(lec04 で投入、Sheets 互換)
- `=CORREL(x_range, y_range)` — 相関係数 r (-1〜+1)
- `=SLOPE(y_range, x_range)` — 最小二乗法の傾き a (引数順序に注意: Y, X の順)
- `=INTERCEPT(y_range, x_range)` — 最小二乗法の切片 b (同上)
- `=RSQ(y_range, x_range)` — 決定係数 R²

これらは `np.polyfit(x, y, 1)` の中身と等価。 「機械が一発で線を引く」 体験を演出するキー関数。

### 避ける関数 (Sheets 古い版で `#NAME?` エラー)
- `IFS` `XLOOKUP` `LAMBDA` `LET` `FILTER` → `IF` の入れ子で代用

## 演習版 + 答え版 の二段階生成 (lec04 で確立)

ビルダースクリプトに `full` パラメータを持たせて、 同じソースから 2 ファイル生成する:

| ファイル | full | 内容 | 配布先 |
|---|---|---|---|
| `情報数理入門_第N回.xlsx` | `False` | サンプル式 1 行のみ・残り空欄黄色、 a/b 等の入力セルは初期値(例: `a=5, b=0`)から開始 | 生徒に配布 |
| `情報数理入門_第N回_答え.xlsx` | `True` | 全行に式入り、 入力セルも正解値(例: `a=6.5, b=−52`)で開始 | 教師用(検証・採点・授業準備) |

ビルダースクリプトの構造:
```python
def sheet_N(wb, full):
    # 共通レイアウト + データ
    for i, (x, y) in enumerate(data):
        if i == 0 or full:
            yel_formula(ws, f'D{r}', f'=式')   # 答え版は全行入り
        else:
            yel_blank(ws, f'D{r}')              # 演習版は 1 行目以外空欄黄色

# main で 2 ファイル生成
for full, suffix in [(False, ''), (True, '_答え')]:
    wb = Workbook()
    for sheet_fn in SHEETS:
        sheet_fn(wb, full)
    wb.save(f'{OUT_DIR}/情報数理入門_第N回{suffix}.xlsx')
```

検証時は答え版を `data_only=True` で評価して、 主要セルの値が期待通りか確認(全行式入りのため `MAX(D10:D28)` や `INDEX/MATCH` も正しく動く)。

## 設問のフィット感 + データ設計の数値美 (lec04 で強化)

問題文と答えが綺麗に成立するよう、 **初期値や定数を逆算して調整する**。

### lec03 例
- 演習②_応用 Q3「配達半径 r で 7 件配達可能」 → r = 5 で**ちょうど 7 件**になるよう住所の座標を設計
- 演習②_応用 Q4「認証する顔 (52, 24, 15, 21)」 → 登録 Sato (50, 22, 14, 20) との距離が **3.16** で「OK」になるよう設計

### lec04 例 (回帰直線で SLOPE / INTERCEPT が綺麗な整数になる設計)

「機械が一発で線を引く」 を体感する演習では、 SLOPE / INTERCEPT の答えが **綺麗な整数** になるようデータを逆算設計する。

**設計手順**:
1. ベースライン直線を決める(例: `y = 35 x + 500`)
2. x 値を等間隔で配置(例: `30, 40, 50, ..., 100`)
3. 各 y_i に **ノイズ ±50, ±100 等** を付加
4. **ノイズ合計 = 0** を保つ → y の平均が `35 × x_avg + 500` のまま
5. **`Σ((x_i − x̄) × noise_i) = 0`** を保つ → SLOPE が 35 のまま不変

**lec04 6_最小二乗法 のデータ実例**:
```
HOUSE = [(30, 1500), (40, 1950), (50, 2150), (60, 2700),
         (70, 3000), (80, 3250), (90, 3750), (100, 3900)]
ノイズ: -50, +50, -100, +100, +50, -50, +100, -100
ノイズ合計 = 0、 Σ(x-65)*noise = 0 を満たす
→ SLOPE = 35 (整数), INTERCEPT = 500 (整数), R² ≈ 0.99 (自然なノイズ感)
80 m² 予測 = 35 × 80 + 500 = 3300 (綺麗)
```

**完全線形にするか、 ノイズあり R² ≈ 0.99 にするかの判断**:
- 「機械学習の威力」 を強調するなら R² < 1 (ノイズあり)。 「機械の方が必ず人間より上」 と言える
- 「数値の整合性」 を最優先するなら R² = 1 (完全線形)。 ただし「予測の必要性」 が薄れる
- lec04 6_最小二乗法 は前者(ノイズあり SLOPE 整数)、 挑戦② は後者(完全線形 R² = 1.0) でメリハリ
