関数とは、
入力に出力を返す決まり。

x という値を入れると、 決まった y を返す 「箱」のような決まり — それが関数。 直線も放物線も sin もすべて関数の仲間。 まずはこの章で、 関数の動きをスライダーで触り、 グラフで見る感覚を取り戻す。

動かしてみる ／ 関数 = 入出力マシン

x を入れると、
決まった y が出てくる箱。

関数は 「入力 x」 を受け取って、 決まった規則で「出力 y」 を返す箱。 下のスライダーで x を動かすと、 青い入力ボールと赤い出力ボールが同時に動く。 関数を切り替えると、 同じ x でも違う y が返ってくる。

同じ関数でもグラフの形は大きく異なる。 直線(2 x、 x+3)、 U 字(x²)、 V 字(|x|)、 階段ではない曲線(√x、 1/x、 sin)、 折れ線(ReLU)。 これらすべてが「入力 → 出力」 の規則という意味で関数の仲間である。
考え方 ／ 関数の見分け方

関数か、 関数じゃないか
縦線テストで見分ける。

関数の定義は「1 つの入力 x には、 出力 y がただ 1 つ決まる」。 グラフ上で 縦線を引くと、 関数なら必ず 1 点で交わる。 もし 2 点以上で交わるなら、 それは関数ではない。

縦線の位置 x1.5
○ 関数: y = x ²
x = 1.5 のとき、 縦線と曲線の交点
1 個 → 関数 ○
✗ 関数じゃない: y² = x
x = 1.5 のとき、 縦線と曲線の交点
2 個 → 関数じゃない ✗
◎
y² = x は x = 4 のとき y = +2 か y = -2 か 1 つに決まらない。 これでは「自販機のボタン」 として使えない (押した結果が 2 つあると困る)。 だから関数ではない。 一方 y = x² は x が決まれば y = x² で 1 つに決まるので関数。

「ズレ」 を測る、
絶対値関数。

|x| は「x が正でも負でも、 ゼロからの距離」 を返す関数。 グラフは V 字になる。 予測と実測のズレを「絶対値」 で測れば、 機械学習や金融で広く使う MAE (平均絶対誤差) が見えてくる。

現象 ／ 「ズレ」 を一つの数で表す

正でも負でも、
絶対値はゼロからの距離。

SCENARIO ／ DISTANCE FROM ZERO
|-3| = 3、 |+3| = 3
符号を取り除いて 大きさだけを残す。 これが 絶対値。 V 字のグラフは、 マイナスの入力もプラスに「折り返して」 返す。
身の回りの例
① 体温の正常値 (36.5℃) からのズレ: +0.8℃ も -0.8℃ も「0.8℃ ずれている」
② 株価の予測値からの誤差: プラス側もマイナス側も「外れ幅」 として等しく評価したい
③ バスケのフリースローのズレ: リングの左 5 cm も右 5 cm も同じ「外し幅」
—— 「正負を気にせず大きさだけ見たい」 場面で、 絶対値を使う。
◎
絶対値 |x| の定義: x ≥ 0 のとき x、 x < 0 のとき -x。 2 通りの場合分けで書かれるが、 グラフは 1 本の V 字で表せる。 V の角 (頂点) は (0, 0) で、 ここで関数の傾きが切り替わる。

滑らかに曲がる、
1/x と √x。

反比例 y = 1/x は速さと時間の関係。 平方根 y = √x はリスク = √分散。 どちらも曲線だが、 直線・絶対値・床関数とは違う 「滑らかな曲がり方」を持つ。

動かしてみる ／ y = k / x

速さを上げるほど、
時間は短くなる。

1000 km の距離を時速 v で走ると、 かかる時間は t ＝ 1000 / v 時間。 速さを 2 倍にすると時間は半分。 これが 反比例 y ＝ k / x の関係。 グラフは 双曲線と呼ばれる滑らかな曲線で、 x ＝ 0 で発散する。

「○ × ○ ＝ 一定」 という関係はすべて反比例。 速さ × 時間、 人数 × 1 人あたり、 圧力 × 体積 (ボイルの法則)。 x が小さくなるほど y は急激に大きくなり、 x → 0 で 「無限大に発散する (定義域外)」のが反比例の特徴。
動かしてみる ／ y = √x

√x は 2 乗して x になる数。 √4 ＝ 2、 √9 ＝ 3。 x が増えると y も増えるが、 増え方は次第に緩くなる。 これは反比例とは違う「もう一つの曲がり方」 で、 統計や金融で 「リスク = √分散」として登場する。

金融への応用 ― リスク ＝ √分散。 株式投資の ボラティリティは、 リターンの分散の平方根。 投資期間を 4 倍にしても、 リスクは 2 倍にしかならない (時間分散の効果)。 これは √x が「ゆっくり増える」性質の応用。
◎
定義域に注意 — √x は x ≥ 0 のときだけ定義される。 x < 0 では実数の答えがない (虚数になる)。 1/x が「x ＝ 0 で発散」 するのに対し、 √x は「x < 0 で存在しない」。 関数ごとに 使える範囲 (定義域)は違う。

AI で使う、
ReLU と シグモイド。

画像認識・音声認識・大規模言語モデル — 現代の AI の中身は 関数の組み合わせ。 中でも ReLU と シグモイド は最も使われる 2 つ。 どちらも今日の絶対値・床関数の親戚。

動かしてみる ／ ReLU(x) ＝ max(0, x)

ReLU は、
絶対値の 右半分。

ReLU (Rectified Linear Unit) は「マイナスは 0 に、 プラスはそのまま返す」 関数。 ChatGPT・画像認識・音声認識など、 現代の AI の中身で 最もよく使われる関数。 グラフは「半分の V 字」 で、 x ＝ 0 で折れ曲がる。

入力 x1.5

入力 x

◎
絶対値と ReLU の関係 ― | x | ＝ ReLU(x) + ReLU(-x)。 つまり 絶対値は ReLU 2 つの組み合わせで書ける。 これは AI の中で「絶対値」 を直接使わず「ReLU の組合せ」 で書くテクニックの基礎。
◎
AI で ReLU が好まれる理由 ― ① 計算が速い (max を取るだけ) ／ ② x ≥ 0 では傾きが 1 のまま (信号が減衰しない) ／ ③ x < 0 では 0 で、 不要な信号を切り捨てる。 これが 「神経細胞の発火」を模した動きで、 ディープラーニングが急速に発展した原動力の一つになった。
動かしてみる ／ σ(x) = 1 / (1 + e^(−x))

シグモイドは、
確率に変換する関数。

シグモイドは どんな実数を入れても 0〜1 の値を返す S 字曲線。 σ(0) ＝ 0.5、 大きな正の入力で 1 に、 大きな負の入力で 0 に近づく。 だから「確率」 として読める。 信用スコア、 病気判定、 メール spam 判定など、 「Yes か No か」 の確率を出したい場面で使う。

入力 x0.0
σ(x) ＝ 1 ÷ ( 1 ＋ e −x ) ／ σ(0) ＝ 0.5、 σ(+∞) → 1、 σ(−∞) → 0
入力 x (スコア)
0.0
σ(x) (Yes である確率)
0.500
判定 (しきい値 0.5)

◎
金融への応用 ― 銀行の 融資審査スコアは、 年収・勤続年数・既存借入などから「貸し倒れスコア」 を計算し、 シグモイドで 0〜1 の貸し倒れ確率に変換する。 0.5 を超えれば「危険」、 下回れば「安全」 のように使う。 同じ構造で、 メールの spam 判定、 病気の有無の判定、 顔認証の本人/別人判定が動いている。

シグモイドの形 ― 中心 (x ＝ 0) 付近で 傾きが最大、 両端では 飽和する (傾きが小さくなる)。 「あいまいな所では敏感に、 はっきりした所では鈍く」 反応するこの性質が、 確率の感覚に合う。 第 2 回の指数関数 ex が分母に入っていることに注目。

FLOW ／ 今日の流れ
01

関数とは、 入力に出力を 1 つだけ返す決まり

関数は 入出力マシンとして理解できる。 同じ入力には必ず同じ出力が返る (縦線テストで 1 点交差)。 直線・放物線・指数・sin・絶対値・床関数・1/x・√x・ReLU・シグモイド — すべてこの定義を満たす関数の家族である。

02

絶対値 |x| はゼロからの距離 — V 字の関数

正でも負でも符号を取り除き、 「ズレの大きさ」だけを返す。 機械学習・金融の 平均絶対誤差(MAE)に直接使われ、 ReLU 2 つの和 ( |x| ＝ ReLU(x)+ReLU(-x) ) でも書ける。 角ばった V 字は、 滑らかな曲線(2 次関数)とは違う 「折れた関数」の代表。

03

同じ「入力 → 出力」 の枠組みが、 AI まで貫く

階段(床関数)・反比例(1/x)・平方根(√x)・ReLU・シグモイドは、 形こそ違うが すべて関数。 AI の中身は ReLU やシグモイドを 大量に組み合わせた巨大な関数でしかない。 第 2 回〜第 6 回で扱った関数たちが、 現代のデータサイエンスの基礎を成している。
